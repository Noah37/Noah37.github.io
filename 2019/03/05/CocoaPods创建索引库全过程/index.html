【macOS】CocoaPods创建索引库全过程

一、序言

Git索引库：顾名思义就是Git仓库的索引表，仓库中的git的项目都包含在列表中。
在终端中输入：

pod repo

可以看到如下内容：

第一项为我即将创建的索引库，第二项则为github的索引库。由于github的私有库收费，我将建立公有库进行示例。

二、正文

1、终端中进入到准备创建示例工程的目录中，本次我们准备创建的工程名为CJModule_A

在终端中输入：
pod lib create CJModule_A

该操作会将github中的pod模板clone到本地，并创建CJModule_A的模块工程

2、创建完成后本地存在一个TestLib的文件夹，我们使用tree命令来查看这个目录的结构。如果报错找不到tree命令，可以采用brew install tree安装。

在 TestLib/Classes/ 文件夹下存放CJModule_A模块的源文件，删除其中的 ReplaceMe.m 文件。

我们在终端中输入

cd Example

进入该模块的项目文件目录，然后输入

pod install

获取最新版本的代码。至此，本地工程创建完成。

三、创建远程CJModule_A工程并链接


1、我们打开github，new一个repo

注意：Initialize README不要勾选，.gitignore与license都选择默认的none，否则待会上传代码会有冲突。

创建完成后获取该github项目的https地址:

https://github.com/NoryCao/TestLib.git

####2、回到本地，编辑TestLib/TestLib.podspec文件

如上，有几个地方需要注意

①. s.description需要填写，否则后面会报警告

②. s.source，s.homepage 需要修改为前面获取的地址
https://github.com/NoryCao/TestLib.git

③. s.resource_bundles的注释打开，这个后面会说到

④. 默认支持的版本为 iOS 8.0以上

⑤. s.summary需要填写，否则后面会报警告

More：http://guides.cocoapods.org/syntax/podspec.html

其他的选项可根据需要填写

3、对本地的podspec文件进行验证

podspec文件的第一行不知道大家注意了没

Be sure to run pod lib lint TestLib.podspec' to ensure this is a valid spec before submitting.

这就是我们要进行的验证：

①. 回到终端，回到TestLib工程的根目录下,输入

pod lib lint TestLib.podspec --private

②. 打包framwork库

通过验证后即可进行打包，这里推荐使用cocoapods的插件cocoapods-packager来进行。

sudo gem install cocoapods-packager

然后终端输入：

pod package TestLib.podspec --force

默认是打包的framwork，如果想要静态库，可在以上命令后添加--library

####4、提交代码到github远程仓库

git 提交代码的基本步骤，就不多说了。

如果本地已经配置过github账号，可继续。若没有配置过，请搜索github账号配置教程，配置完成后继续以下步骤。

第一行：将本地的工程与远程工程关联起来

第二行：将本地代码推送到远程分支

5、验证远程podspec

验证远程podspec前需要先打tag，并且该tag需要与podspec中的s.version一致

6、添加远程索引库

与前面创建远程仓库方法一致，除名称外，其他默认即可。

然后添加本地索引库并与远程库进行关联，成功后推送到远程。

以上操作成功后，本地索引库如下

pod spec

到这里，索引库与CJModule_A工程都已经成功创建并上传github，这里我们来验证下，在终端输入

pod search TestLib

可搜索到以下内容，版本0.1.3是我制作第二次产生的，后面会讲到

7、创建主工程MainProject并引入TestLib

终端中进入到MainProject根目录下，输入

pod init

在生成的Podfile文件添加source

然后执行
pod install

完成后，TestLib框架已被引入到MainProject中

8、为TestLib添加资源文件

1、我们来为TestLib添加资源文件，本次示例添加图片

usercenter_nav_setting.png

前面打开注释的podspec中的代码段：

s.source_files匹配源代码文件

s.resource_bundles匹配资源文件

如果是非png文件，可将后缀改为*.*

2、重新发布版本

  • 修改podspec文件的s.version为0.1.3
  • 验证本地podspec文件pod lib lint --private

  • 提交代码

  • 打tag

  • 打包

  • 打完包后再次提交代码

  • 推送cjspecs
